# 7 Pillars Feature - Complete Functional & Features Guide

## Overview

The 7 Pillars is a governance and community contribution framework that transforms the platform from a simple discussion forum into a structured knowledge-building system. It organizes all discussions around 6 foundational pillars, where community members collectively answer fundamental questions about how the platform should function.

**Core Concept**: Each pillar represents a core value, and within each pillar are 7 "seed questions" that the community collectively works to resolve through discussion and consensus.

## Core Features & User Experience

### 1. **Pillar Discovery Hub**
**What Users See**: A main hub page showing all 6 pillars as beautiful, clickable cards.

**Features**:
- **Overall Platform Progress**: A progress bar showing what percentage of all seed questions have been "resolved" or actively discussed
- **Pillar Cards**: Each pillar displays:
  - Pillar icon and name (with Latin name subtitle)
  - Short description
  - Progress bar (0-100% completion)
  - Status badge: "üÜï New", "üí¨ In Progress", "üî• Active", "‚úì Complete"
  - Call-to-action: "Explore ‚Üí"
- **Recent Achievements Section**: Milestones the community has hit (e.g., "Freedom pillar reached 50% completion")
- **Next Milestone Preview**: What's the next goal the community is working toward?

### 2. **Individual Pillar Pages**
**What Users See**: Deep dive into a specific pillar.

**Features**:
- **Pillar Header**: Full description, icon, color-coded theme
- **Seed Questions List** (7 per pillar):
  - Question text
  - Current discussion status (Hot, New, Active, Converging, Resolved)
  - Metrics: Number of posts, participants, activity level
  - "Join Discussion" button that takes them to the actual discussion
- **Sorting Options**:
  - **Hot**: Most active discussions (engagement-weighted)
  - **New**: Recently started discussions
  - **Participants**: Most people involved
  - **My Activity**: Discussions the user has participated in (auth required)
- **Progress Tracking**: How many of the 7 questions in this pillar have been resolved

### 3. **Discussions Hub Integration**
**What Users See**: When browsing all discussions, users can filter by pillar.

**Features**:
- **Pillar Tabs/Filter**: A row of pillar buttons (6 tabs) to filter discussions
- **Dynamic Discussion Feed**: Shows only discussions tagged to that pillar
- **Sorting Within Pillar**: Apply same sorting (hot, new, participants, my activity)
- **Visual Consistency**: Each pillar's color theme carries through (red for Freedom, purple for Justice, etc.)

### 4. **Discussion Status System**
**How It Works**: Each discussion gets a status based on activity level and community engagement.

**Status Types**:
- **üÜï New**: Just started, <5 posts, <3 participants
- **üí¨ In Progress**: Actively being discussed, 5-20 posts
- **üî• Hot**: Trending, high engagement, many posts this week
- **‚öñÔ∏è Converging**: Heading toward consensus, slowing posts but good participation
- **‚úì Resolved**: Community has reached consensus or discussion has concluded

**Heat Score** (0-100): Combines:
- Recency (recent posts boost score)
- Weekly velocity (posts per week)
- Participant count (more voices = hotter)
- Total post volume

## User Interaction Flows

### Flow 1: Browse & Explore
1. User arrives at `/pillars` (Pillars Hub)
2. Sees all 6 pillars with progress cards
3. Clicks on a pillar (e.g., "Freedom")
4. Routes to `/pillars/freedom`

1. When a pillar reaches 50%, community celebration notification
2. When a pillar reaches 100%, special achievement badge/announcement

## Content Moderation & Resolution Logic

### How Questions Get "Resolved"
A seed question is marked as resolved when:
1. **Discussion has stabilized**: Post volume has dropped (consensus reached or debate exhausted)
2. **Community consensus indicator**: Admin or moderation team reviews discussion and identifies emerging consensus
3. **Timeline**: After a certain period of activity (e.g., 30+ days of discussion)
4. **Manual marking**: Moderators can mark as resolved if clear consensus exists

### Reopening Questions
- If new evidence or perspectives emerge, a resolved question can be reopened
- Community can vote to reopen
- Keeps the system dynamic and responsive to new information

- Main seed questions have sub-questions (optional depth)
