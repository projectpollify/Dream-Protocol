Enforcement Mechanisms

1. Smart Contracts
PollCoin and Gratium contracts explicitly disable lending integrations. Tokens cannot be used as collateral. Transfer hooks prevent sending to blacklisted contracts.

2. DEX Monitoring
Watch Uniswap, Sushiswap, and other DEXes 24/7. Alert system triggers on any leverage/shorting detection. Community can report violations for Light Score rewards.

3. Governance Penalties
If caught violating spot-only, user is banned from governance permanently. This creates strong incentive to comply.

üìä Database Schema Overview

5 Core Tables

1. token_purchase_orders

Tracks all fiat-to-token purchases

CREATE TABLE token_purchase_orders ( id UUID PRIMARY KEY, user_id UUID, identity_mode VARCHAR(10), -- 'true_self' or 'shadow' fiat_amount DECIMAL, fiat_currency VARCHAR(3), -- 'USD', 'EUR', etc token_type VARCHAR(10), -- 'pollcoin' or 'gratium' token_amount BIGINT, price_per_token DECIMAL, premium_percentage DECIMAL, -- Usually 15% payment_provider VARCHAR(50), -- 'stripe', 'moonpay', 'wyre' status VARCHAR(20), -- 'pending', 'completed', 'failed', etc kyc_verified BOOLEAN, created_at TIMESTAMPTZ, completed_at TIMESTAMPTZ );

2. dex_listings

Monitor external DEX listings for compliance

CREATE TABLE dex_listings ( id UUID PRIMARY KEY, dex_name VARCHAR(50), -- 'uniswap', 'sushiswap' token_type VARCHAR(10), pool_address VARCHAR(66), trading_pair VARCHAR(30), -- 'POLL/USDC' liquidity_usd DECIMAL, price_usd DECIMAL, volume_24h DECIMAL, is_leverage_pool BOOLEAN, is_shorting_enabled BOOLEAN, has_lending_integration BOOLEAN, compliance_status VARCHAR(20), -- 'compliant', 'violation_detected' created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ );

3. purchase_limits

Enforce tier-based purchase limits

CREATE TABLE purchase_limits ( id UUID PRIMARY KEY, user_id UUID UNIQUE, verification_tier VARCHAR(20), -- 'unverified', 'basic', 'verified', etc daily_limit BIGINT, monthly_limit BIGINT, yearly_limit BIGINT, purchased_today BIGINT DEFAULT 0, purchased_this_month BIGINT DEFAULT 0, purchased_this_year BIGINT DEFAULT 0, last_purchase_at TIMESTAMPTZ, last_reset_daily TIMESTAMPTZ );

4. price_history

Historical pricing for analytics

CREATE TABLE price_history ( id UUID PRIMARY KEY, token_type VARCHAR(10), source VARCHAR(50), -- 'on_platform', 'uniswap', etc price_usd DECIMAL, volume_24h DECIMAL, market_cap DECIMAL, platform_sales_24h BIGINT, platform_revenue_24h DECIMAL, recorded_at TIMESTAMPTZ );

5. compliance_alerts

Track spot-only violations and concerns

CREATE TABLE compliance_alerts ( id UUID PRIMARY KEY, alert_type VARCHAR(50), -- 'lending_detected', 'shorting_enabled', etc severity VARCHAR(20), -- 'info', 'warning', 'critical' token_type VARCHAR(10), dex_name VARCHAR(50), description TEXT, recommended_action TEXT, status VARCHAR(20), -- 'open', 'investigating', 'resolved' created_at TIMESTAMPTZ, resolved_at TIMESTAMPTZ );
