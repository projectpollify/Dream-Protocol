üîÑ Complete Vote Flow

Step 1: Poll Creation

Cost: 500-1000 PollCoin (depends on poll type)
Who: Verified humans only (PoH 50+, Light Score 20+)
What happens: 1% burned, 99% goes to rewards pool

Step 2: Voting Period

Duration: 7-14 days (configurable)
Who can vote: Verified humans only
Voting: Both identities vote (can be same or opposite)

Step 3: 7-Section Assignment

What: System randomly assigns voters to 7 sections
Why: Prevents whale domination
Result: Each section gets random 0.7x-1.5x multiplier

Step 4: Weighted Voting

Calculation: Base votes √ó section multiplier = weighted votes
Whale protection: Spreads their votes across sections
Fairness: Smaller holders' votes matter more on average

Step 5: Shadow Consensus

True Self tally: % voting YES
Shadow tally: % voting YES
Gap: |True Self % - Shadow %|
Confidence: Statistical margin of error calculated

Step 6: Decision

If >50% YES: Proposal approved
If ‚â§50% YES: Proposal rejected
Wait: 72 hours before execution (rollback window)

Step 7: Execution (if approved)

What: Governance action takes effect
Where: Recorded on Arweave (permanent)
Rollback: Can be reverted within 72 hours with new vote

Key Design Decisions

Separate identity_mode tracking: True Self and Shadow are tracked as separate voting entities but linked by user_id (which stays private).

7-section assignment: Random section multipliers prevent gaming; stored per-vote for audit trail.

Shadow Consensus snapshots: Calculated immediately after poll closes and stored for historical analysis.
